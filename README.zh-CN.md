# clock-in

**把你的 Claude Code 会话，开成一家你说了算的公司。**

CEO 负责拆解和调度，各部门动手干活，独立审查官不放行就不算完成，还有一块实时面板：所有等你拍板的事，都在上面。

## 它能做什么

- **让「完成」名副其实。** 计划要过审，产出也要过审：独立审查官没放行之前，任何任务都不算完成。质量来自结构，而不是指望一个模型给自己批作业。
- **所有需要你的事，集中在一处。** 每条请示带着当场做决定所需的信息落到 Boss 面板上：问题、选项、推荐。请示里提到的文件一键就能打开，哪怕它还躺在某个部门没合并的 worktree 里。再也没有滚过屏幕就错过的消息。
- **看得懂圈圈画画的截图。** 把出问题的地方圈出来，贴进来。CEO 先逐条复述它理解到的标记，确认无误才派工：读歪了只花你两行字，而不是一轮白干。然后各部门拿到一张「可能原因」清单，每条原因配一个验证方法和确认后的修法。原因没验证不许动手修，证据要写进汇报。
- **跟真实任务系统对齐。** 任务卡由 Claude Code 自带的任务系统生成，完工自动退场，看板不会悄悄跟现实脱节。手写的、没有登记的卡，下次开工时会被点名。
- **卡住的活儿早发现。** 反复被打回的任务会被拉出返工循环做诊断；诊断之后还卡着，就变成一条摆到你面前的决策，而不是一个惊喜。
- **有共享记忆。** 决策和定论写在每个部门都会读的地方：项目再大，定了的事也不会被重新吵一遍，或者被悄悄推翻。
- **有人盯着看门人。** 独立督察审计整个团队，包括 CEO 自己，只对你汇报。
- **聪明模型花在判断上，便宜模型花在打字上。** Opus 当 CEO 时，部门主管跑 Opus，他们雇的员工跑 Sonnet，纯体力活给 Haiku。换 Fable 上场，整个组织围着它重排（见下文）。
- **自己收拾自己。** 部门为任务而生，交付即退场，不会攒出一排僵尸窗格。干完活闷声不响的，会收到一次机械提醒去交报告。纪律绊线（状态文件超标、请示写成小作文、没登记的卡）开工时各提示一行，一切干净时零 token。要紧的留在眼前，其余落盘：长项目不臃肿、不丢线索。
- **拦得住毁灭性命令。** 强推、递归删除、删表，落地之前就被截下。

## 这家公司

你是 **Boss**（董事长）。你的 Claude Code 会话变成 **CEO**：把目标拆成任务，逐个交给**部门**（研发、测试、运维、法务、财务、文档，活儿需要什么就有什么）。独立的**审查官**在开工前把关每份计划、交付前把关每份产出。**督察**监督整个公司，包括 CEO，直接向你汇报。

市面上大多数「多智能体」，是同一个提示词轮流换帽子。clock-in 建立在**分权**上：角色之间真的互相制衡，而且这些检查自动触发，不靠任何人记得去问。

## 同一句口令，两种公司

组织结构会适配坐在驾驶座上的模型。

**Opus 掌舵（默认）。** 判断力是分布式的：CEO 负责路由和排期，不插手手艺；每个部门主管自己决定活儿怎么干，并审查自己雇的员工。

**Fable 掌舵。** 会话变成大脑。它翻译你的意思，对着你圈过的截图直接诊断，一个文件都不用翻，然后把「原因、验证、修法」表格交给 Sonnet 部门去确认执行。贵的模型只握着文字、表格和汇报，从不碰代码：你的 Fable 额度花在思考上，而不是读它根本不需要读的文件。

同一句口令，同样的门禁，同一块面板。你用什么模型开工，就召唤出哪一家公司。

## 怎么用

想要什么，用大白话说。CEO 起草计划先给你过目；你点头之后各部门开工、互相把关，最后你收到一份简短清晰的变更汇报。

       alt="各部门把完成的工作向上汇报给 CEO，等待独立复核。">
</p>
<p align="center"><sub>公司运转中：部门向上汇报，每份产出都要过独立审查才算数。</sub></p>

你永远不会被隔在外面。创始人模式意味着你可以随时空降任何部门，当面把事情聊透，CEO 事后自动跟进。你在部门窗格里的时候，CEO 保持安静、不打扰你们；你一离开，部门就汇报改了什么，CEO 只从汇报同步。就算工作量大到你脑子装不下，控制权依然全在你手里。

挂机跑一晚，回来看到的是一份一眼扫完的摘要：**发布了什么、排着什么、什么在等你拍板**，而不是一场考古挖掘。

## 快速开始

### 你需要

- **开启了 Agent Teams 的 Claude Code。** 整套东西跑在 teammates 上。
- **Python 3。** 只用标准库，不用 `pip install` 任何东西。
- **一个 git 仓库**（推荐）。团队边干边提交，历史随时可查。

在 Claude Code 的 `settings.json` 里开启 Agent Teams：

```json
{ "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
```

### 安装

```text
/plugin marketplace add Lumos221/clock-in
/plugin install clock-in@mycompany
```

然后重启 Claude Code。启用即自动接线，没有额外配置。

### 开工

打开一个项目，说一句 **「开始上班」**（或 "clocking in"）。第一次它会组建这个项目需要的团队，并请你重启一次让配置生效；之后每次都接着上次继续。

> [!TIP]
> **小事别开公司。** 改一个文件不需要一家公司，直接问 Claude 就好。多线并行的活儿，clock-in 才回本。

## 为什么做它

我的念头又跳又碎，开的头比收的尾多，不写下来的就丢了，所以我特别依赖脑子外面的结构。clock-in 就是那套结构在 AI 智能体上的翻版：制衡、共享记忆、以及一个所有等我的事都会浮上来的地方。

为需要结构才能走直线的脑子设计，得到的是对所有人都更稳的东西。
